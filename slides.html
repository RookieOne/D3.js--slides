Data drives DOM

D3 allows you to bind arbitrary data to a Document Object Model (DOM), and then apply data-driven transformations to the document.

Source: d3js.org

Unbound Creativity

D3 is not a monolithic framework that seeks to provide every conceivable feature. Instead, D3 solves the crux of the problem: efficient manipulation of documents based on data.

Source: d3js.org

D3 is more than a charting library

Add SVG 

    var g = d3.select("#bar-chart")
            .append("svg:svg")
            .attr("width", width + margin)
            .attr("height", height + margin)
            .append("svg:g");

Set linear X scale

    var x = d3.scale.linear()
        .domain([0, 100])
        .range([min, max]);
        
    var range_value = x(domain_value);
    var pixel_x = x(domain_value);

Set ordinal Y scale

    var y = d3.scale.ordinal()
        .domain([categories])
        .range([5,7,9,13,23]);
        
    var range_value = y(category);
    var pixel_y = y(category);

Add bars

    g.selectAll(".bars")
        .data(data)
        .enter()
        .append("svg:rect")
        .attr("x", x(0))
        .attr("y", function(d, i) {
            return y(i);
        })
        .attr("width", function(d) {
            return x(d.value) - margin;
        })
        .attr("height", barHeight);

Add classes

    g.selectAll(".bars")

X-Labels

    // x-labels
    g.selectAll("x-label")
        .data(x.ticks(5))
        .enter()
        .append("svg:text")
        .attr("class", "x-label")
        .text(function(d) {
            return d;
        })
        .attr("x", function(d) {
            return x(d);
        })
        .attr("y", height + 25)
        .attr("text-anchor", "middle");

Bar Labels

    // bar labels
    g.selectAll("bar-label")
        .data(data)
        .enter()
        .append("svg:text")
        .text(function(d) {
            return d.stat;
        })
        .attr("class", "bar-label")
        .attr("x", function(d) {
            return 35;

Set Opacity to States

    var opacity = d3.scale.linear().domain([0,100]).range([.5, 1]);
    _.each(data, function(d) {
        d3.selectAll("#" + d.state)
                .attr("style", "")
                .attr("class", "default-style")
                .attr("opacity", opacity(d.value))
                .attr("fill", "#0000FF");
    });

Set Color to States

    var color = d3.scale.linear().domain([0,100]).range(["#000000", "#0000FF"]);
    _.each(data, function(d) {
        d3.selectAll("#" + d.state)
                .attr("style", "")
                .attr("class", "default-style")
                .attr("opacity", 1)
                .attr("fill", color(d.value));
    });

Final Comments

80/20

80% of the chart will be done in 20% of the time.

The last 20% of work getting the chart awesome will take 80% of your time.

Good Workflow

  1. Create visual with mock data in separate html file
  2. Finalize data format
  3. Test your components output data in correct format
  4. Wire your data to your visual

Lines of Code

  • D3 can result in a large number of lines of code
  • But the code is simple once you understand the pattern.

Yes I can do that

Isn't it nice telling clients and designers yes?

Other charting libraries might get a chart out faster but few can compete with d3's flexibility.
